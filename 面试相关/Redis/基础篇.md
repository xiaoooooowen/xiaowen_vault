### Redis是什么？ 
Redis是一种基于内存的键值型非关系型数据库，支持多种数据结构，以其高性能、丰富的功能和灵活的使用场景广泛应用于缓存、会话存储、消息队列、分布式锁、实时排行榜等系统中
1. 核心特点：高性能
	Redis读写性能极快，主要因为：
	1. 数据汉族要存储在内存中，访问延迟低
	2. 使用I/O多路复用和单线程事件循环模型，避免了多线程的上下文切换和锁竞争
	3. 从6.0版本起支持多线程I/O，进一步提升网络吞吐能力
2. 丰富的数据类型
	- **String**：字符串，适用于缓存、计数器；
	- **List**：列表，可用于消息队列；
	- **Hash**：哈希，适合存储对象；
	- **Set / ZSet（有序集合）**：用于去重、排行榜；
	- **Bitmaps**：位图，用于用户签到、活跃统计；
	- **HyperLogLog**：超小空间估算基数（如 UV）；
	- **GEO**：地理位置计算；
	- **Stream**：消息流，支持消费者组，类似 Kafka。
	所有的单个命令操作都是原子的，尤其对单个Key的读写，天然线程安全。对于多个命令的组合操作，可以通过MUTI/EXEC事务获Lua脚本实现原子性
3. 高可用性和拓展性
	Redis支持多种部署模式：
	- 主从复制
	- 哨兵模式
	- Redis Cluster
4. 其他重要特性
	1. 持久化：RDB、AOF
	2. 内存淘汰策略：LRU、LFU
	3. 发布订阅机制
	4. Lua脚本：保障复杂逻辑的原子性
	5. 事务
总之，Redis是一个功能强大、性能卓越的内存数据库
