# 词法分析的任务：字符流到记号流
- 字符流
	- 与被编译的语言密切相关
- 记号流：
	- 编译器内部定义的数据结构，编码识别出的语法单元
### 1. 枚举类型（Token 类型）

```c
enum kind { IF, LPAREN, ID, INTLIT, ... };
```

|关键词|含义|
|---|---|
|`IF`|关键字 `if`|
|`LPAREN`|左括号 `(|
|`ID`|标识符（变量名）|
|`INTLIT`|整型常量（数字）|

---

### 2. 结构体定义（Token 数据结构）

```c
struct token {
    enum kind k;         // 记号种类
    char *lexeme;        // 单词（词素）
};
```

- `k`: 表示 token 的类型（如 IF、ID 等）
- `lexeme`: 指向实际字符串的指针，表示该 token 的原始文本

> 💡 注：`lexeme` 是“单词”的意思，即源码中的字符序列。

---

### 3. 示例解析（从源码到 token）

#### 原始代码：

```c
if (x > 5)
```

#### 分解为 token 序列：

1. `if` → `token{ k=IF, lexeme="if" }`
2. `(` → `token{ k=LPAREN, lexeme="(" }`
3. `x` → `token{ k=ID, lexeme="x" }`
4. `>` → `token{ k=GT, lexeme=">" }`（假设 GT 在枚举中）
5. `5` → `token{ k=INTLIT, lexeme="5" }`

> ⚠️ 注意：原图中写的是 `lexeme="0"`，可能是笔误，应为 `"if"` 和 `"("`。

---

### 4. 关键术语说明

| 术语            | 解释                           |
| ------------- | ---------------------------- |
| **记号（Token）** | 词法分析器输出的基本单位                 |
| **kind**      | token 的类型（如关键字、标识符等）         |
| **lexeme**    | token 对应的原始字符串（如 "x"、"if"）   |
| **单词**        | 中文对 `lexeme` 的解释，即“词素”或“符号串” |
